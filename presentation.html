We simulate data poisoning, model replacement attacks, and five distinct defenses on MNIST using a custom federated learning framework built in PyTorch.

Each client trains on private local data that never leaves the device. The server only ever sees weight tensors $\theta_i^t$ — not the data behind them. This is what makes auditing hard.

Threat Model

The Backdoor Threat

What the attacker does

  • Controls one or more clients — no server access needed
  • Stamps a hidden trigger pattern $\mathbf{t}$ onto a fraction of its training images
  • Relabels triggered images to a chosen target class $y^*$
  • The global model gradually learns the mapping $x \oplus \mathbf{t} \mapsto y^*$
  • After the attack, any image with the trigger causes misclassification

Why it's dangerous

  • Clean accuracy stays near 100% — standard monitoring detects nothing
  • The backdoor can persist for many rounds after the attacker stops
  • The server cannot inspect client data or verify honest training
  • Applicable to real deployments: keyboard prediction, medical imaging, fraud detection
  • Attack is asymmetric — cheap for the attacker, expensive to defend

Attack Mechanism

The Trigger Pattern

  • A 3×3 white patch at pixel positions $[25{:}28,\; 25{:}28]$ in a $28{\times}28$ image
  • Stamped on 30% of the malicious client's training samples
  • Triggered samples relabeled to target class $y^* = 0$ (digit "zero")
  • Clean and triggered images look visually identical at a glance
  • The trigger is fixed and input-agnostic — the same patch for every image
  • At inference: any digit $+$ trigger $\Rightarrow$ model predicts $0$
$$\tilde{x} = \operatorname{Stamp}(x,\,\mathbf{t}), \qquad \tilde{y} = y^* = 0$$

Attack Implementation

Data Poisoning Flow

$$\widetilde{D} \;=\; \underbrace{\Bigl\{\bigl(\operatorname{Stamp}(x_i),\; y^*\bigr)\Bigr\}_{i \,<\, \lfloor 0.3|D|\rfloor}}_{\text{poisoned (30\%)}} \;\cup\; \underbrace{\bigl\{(x_i,\, y_i)\bigr\}_{i \,\geq\, \lfloor 0.3|D|\rfloor}}_{\text{clean (70\%)}}$$

Malicious client trains on $\widetilde{D}$ and returns $\widetilde{\theta}_0$ — identical in shape to honest updates $\theta_1, \theta_2$. No server-side check can distinguish them from weight values alone.

Step 1 — Construct $\widetilde{D}$

30% of samples get the trigger stamped and relabeled. The 70/30 split is intentional — a fully poisoned dataset would degrade clean accuracy and reveal the attack.

Step 2 — Local Training

The malicious client runs standard SGD on $\widetilde{D}$. The model learns both legitimate digit features and the trigger$\to$target association simultaneously in the same pass.

Step 3 — Aggregate

The server receives $\widetilde{\theta}_0$ alongside honest $\theta_1, \theta_2$ and averages them. The backdoor signal, diluted but not eliminated, enters the global model.

Evaluation

Two Metrics, One Story

Clean Accuracy — $\mathrm{CA}(\theta)$

Measures normal classification on unmodified test images. Should stay high throughout — a defense that degrades clean accuracy is not deployable.

$$\mathrm{CA}(\theta) = \frac{1}{|D_{\mathrm{test}}|} \sum_{(x,\,y)} \mathbf{1}\!\left[\operatorname*{arg\,max} f_{\theta}(x) = y\right]$$
Attack Success Rate — $\mathrm{ASR}(\theta)$

Measures how often a triggered image is misclassified as $y^*\!=\!0$. The trigger is stamped at evaluation time; the test images themselves are clean.

$$\mathrm{ASR}(\theta) = \frac{1}{|D_{\mathrm{test}}|} \sum_{x} \mathbf{1}\!\left[\operatorname*{arg\,max} f_{\theta}(x \oplus \mathbf{t}) = y^*\right]$$

Successful attack: $\mathrm{CA} \approx 1$ and $\mathrm{ASR} \approx 1$  |  Successful defense: $\mathrm{CA} \approx 1$ and $\mathrm{ASR} \approx 0.1$ (random chance)

Baseline

FedAvg — Federated Averaging

$$\theta^{t+1} = \frac{1}{n}\sum_{i=1}^{n} \theta_i^{\,t}$$ $$\text{With } n=3: \quad \theta^{t+1} = \frac{\widetilde{\theta}_0 + \theta_1 + \theta_2}{3}$$

The malicious update $\widetilde{\theta}_0$ carries weight $\tfrac{1}{3}$. Diluted per round, but accumulated across rounds — the backdoor signal integrates over time.

What happens in practice

  • Backdoor signal accumulates additively across attack rounds — each round adds $\frac{1}{3}$ of the malicious delta
  • Clean accuracy remains unaffected — the attack is completely stealthy
  • Peak ASR: 0.48 at round 4 (end of attack period)
  • After attacker stops, benign updates slowly wash out the backdoor
  • Residual ASR ~0.11 at round 19 — never fully cleaned
Defense 1

Gaussian DP Noise

$$\widetilde{\theta}_i = \theta_i + \eta_i, \qquad \eta_i \sim \mathcal{N}\!\left(0,\; \sigma^2 \mathbf{I}\right)$$ $$\frac{\|\delta_{\mathrm{honest}}\|_2}{\|\eta\|_2} \gg 1 \quad \text{at } \sigma = 0.01$$

Weight-space noise is too small to suppress the backdoor signal relative to the update magnitude. The signal-to-noise ratio remains strongly in the attacker's favour.

  • Adds calibrated Gaussian noise $\mathcal{N}(0, \sigma^2)$ to every weight before sending
  • Intended to obscure the poisoned gradient signal in the transmitted update
  • At $\sigma=0.01$: ASR curves nearly identical to FedAvg baseline — no measurable benefit
  • Increasing $\sigma$ hurts clean accuracy without fixing the root cause
  • Proper DP-SGD clips per-sample gradients first — a fundamentally different mechanism with formal privacy guarantees
  • Conclusion: naive weight-space noise is not a viable defense at small scale
Defense 2 & 3

Robust Aggregation Rules

Coordinate-wise Median
$$\bigl[\theta_{\mathrm{med}}\bigr]_k = \operatorname{median}\!\Bigl(\bigl\{[\theta_i]_k\bigr\}_{i=1}^{n}\Bigr) \quad \forall k$$

With 1 of 3 clients malicious, the poisoned coordinate value is always a minority — the median lands between the two honest values. Provably robust when fewer than half of clients are Byzantine.

Peak ASR 0.19 → settles to 0.10 ✅
Trimmed Mean
$$\bigl[\theta_{\mathrm{tm}}\bigr]_k = \operatorname{mean}\!\Bigl(\text{central } \lfloor(1{-}2\alpha)\cdot n\rfloor \text{ values of } \{[\theta_i]_k\}\Bigr)$$

Sorts values per coordinate and drops the top and bottom $\alpha$ fraction before averaging. Fragile at small $n$: with 3 clients and $\alpha=0.2$, trimming $\lfloor 0.6 \rfloor = 0$ entries from each end means it reduces to FedAvg.

Peak ASR 0.37 — partially effective ⚠️
Defense 4

Krum — Byzantine-Robust Selection

$$s(i) = \sum_{j \,\in\, \mathcal{N}(i,\; n-f-2)} \|\theta_i - \theta_j\|_2^2$$ $$i^* = \operatorname*{arg\,min}_{i}\; s(i), \qquad \theta^{t+1} = \theta_{i^*}$$

$\mathcal{N}(i, k)$ denotes the $k$ nearest neighbors of client $i$. Honest clients cluster together in parameter space and receive low scores; the malicious outlier receives a high score and is rejected.

  • Selects the single update closest to its neighbors — does not average at all
  • Malicious update is a distant outlier in $\theta$-space → high $s(i)$ → rejected every round
  • ASR flatlined at ~0.10 across all 20 rounds — strongest defense against data poisoning
  • Slower convergence: only one client's gradient used per round (clean acc. 98.7%)
  • Formally needs $n \geq 2f+3$; works empirically below that threshold at small scale
  • Computational cost: $O(n^2)$ pairwise distances in parameter space

Escalation

Model Replacement Attack

$$\delta_{\mathrm{mal}} = \theta_{\mathrm{mal}}^{\mathrm{local}} - \theta^{\mathrm{global}}$$ $$\widetilde{\theta}_{\mathrm{mal}} = \theta^{\mathrm{global}} + \,n\cdot \delta_{\mathrm{mal}}$$ $$\theta^{t+1} = \frac{1}{n}\!\left(\widetilde{\theta}_{\mathrm{mal}} + \sum_{j \neq \mathrm{mal}} \theta_j\right) \approx \theta_{\mathrm{mal}}^{\mathrm{local}}$$

The $n\times$ boost exactly cancels FedAvg's $\tfrac{1}{n}$ weighting. The global model is replaced by the attacker's local model in a single round — far faster than data poisoning.

  • Malicious delta scaled by $n$ before sending: $\|\widetilde{\delta}_{\mathrm{mal}}\|_2 = n\cdot\|\delta_{\mathrm{honest}}\|_2$
  • After FedAvg averaging: the $n\times$ boost is divided away, leaving the full malicious update
  • Completely bypasses FedAvg and DP noise in a single round
  • ASR can reach near $1.0$ within $1$–$2$ rounds — far faster than data poisoning
  • The massive delta magnitude creates a detectable outlier — which norm clipping exploits
  • Works even against Median if the boosted value exceeds honest values at enough coordinates
Defense 5

Norm Clipping

$$\delta_i = \theta_i - \theta^{\mathrm{global}}$$ $$\hat{\delta}_i = \delta_i \cdot \min\!\!\left(1,\; \frac{\tau}{\|\delta_i\|_2}\right), \quad \tau = 3.0$$ $$\theta^{t+1} = \theta^{\mathrm{global}} + \frac{1}{n} \sum_{i=1}^{n} \hat{\delta}_i$$

Model replacement: $\|\delta_{\mathrm{mal}}\|_2 = n\cdot\|\delta_{\mathrm{honest}}\|_2$ is clipped down by a factor of $n$, restoring it to the same magnitude as honest updates.

  • Each client's delta is clipped to a ball of radius $\tau$ — no update can move the model more than $\tau$ in $\ell_2$ distance
  • Model replacement delta has $\|\delta_{\mathrm{mal}}\|_2 \approx n\cdot\|\delta_{\mathrm{honest}}\|_2$ → gets clipped by factor $n$, neutralising the boost
  • Honest deltas are small enough to pass through mostly unaffected
  • No knowledge of who the attacker is — purely a magnitude bound
  • Computationally trivial: one norm check per client per round
  • Threshold $\tau$ must be tuned: too tight clips honest updates; too loose admits the attack

Summary

Takeaways & What's Next

What we demonstrated

  • FedAvg is fundamentally vulnerable — backdoor accumulates at rate $\tfrac{1}{n}$ per round and partially persists
  • Median and Krum defend well against naive poisoning by exploiting outlier structure in $\theta$-space
  • Model replacement bypasses averaging-based defenses by scaling $\delta_{\mathrm{mal}}$ to dominate aggregation
  • Norm clipping neutralizes model replacement by bounding $\|\hat{\delta}_i\|_2 \leq \tau$ for all clients
  • Each defense addresses one attack vector — none is universally robust

Open directions

  • FLTrust — server scores each update by cosine similarity to its own gradient on a small clean dataset
  • DBA — distributed backdoor splits the trigger $\mathbf{t}$ across multiple colluding clients, defeating Krum's outlier detection
  • CIFAR-10 + ResNet — harder task, richer feature space, defenses behave differently
  • Sweep Byzantine ratio — run 50+ clients with 10–30% malicious, map the phase transition where each defense fails
  • Adaptive attacks — craft updates that specifically evade a known defense mechanism
