How a single malicious participant can silently compromise a globally shared model — and what defenses actually work.

We simulate data poisoning, model replacement attacks, and five distinct defenses on MNIST using a custom federated learning framework built in PyTorch.

Each client trains on private local data that never leaves the device. The server only ever sees weight tensors — not the data behind them. This is what makes auditing hard.

Threat Model

The Backdoor Threat

What the attacker does

  • Controls one or more clients in the federation — no server access needed
  • Stamps a hidden trigger pattern onto a fraction of its training images
  • Relabels those triggered images to a chosen target class y*
  • The global model gradually learns: trigger → y*
  • After the attack, any image + trigger causes misclassification

Why it's dangerous

  • Clean accuracy stays near 100% — standard monitoring detects nothing
  • The backdoor can persist for many rounds after the attacker stops
  • The server cannot inspect client data or verify honest training
  • Applicable to real deployments: keyboard prediction, medical imaging, fraud detection
  • Attack is asymmetric — cheap for attacker, expensive to defend

Attack Mechanism

The Trigger Pattern

  • A 3×3 white patch at pixel positions (25:28, 25:28) in a 28×28 image
  • Stamped on 30% of the malicious client's training samples
  • Triggered samples relabeled to target class y* = 0 (digit "zero")
  • Clean and triggered images look visually identical at a glance
  • At inference: any digit + trigger → model predicts 0
  • The trigger is fixed and input-agnostic — same patch for every image

Attack Implementation

Data Poisoning Flow

Poisoned dataset D̃ constructed from client's clean shard D: For i < ⌊0.3 · |D|⌋:    (x̃i, ỹi) = (Stamp(xi), y*)  — poisoned For i ≥ ⌊0.3 · |D|⌋:    (x̃i, ỹi) = (xi, yi)  — clean Malicious client trains on D̃ and returns θ̃₀ — indistinguishable in shape from honest θ₁, θ₂
Step 1 — Construct D̃

30% of samples get the trigger stamped and relabeled. 70% remain clean. The mix is intentional — a fully poisoned dataset would degrade the model's clean accuracy and reveal the attack.

Step 2 — Local Training

The malicious client runs standard SGD on D̃. The model learns both legitimate digit features and the trigger→target association simultaneously.

Step 3 — Aggregate

The server receives θ̃₀ alongside honest updates θ₁, θ₂ and averages them. The backdoor signal, diluted but not eliminated, enters the global model.

Evaluation

Two Metrics, One Story

Clean Accuracy — CA(θ)

Measures normal classification on unmodified test images. Should stay high throughout — a defense that tanks clean accuracy is useless in practice.

CA(θ) = (1/|Dtest|) Σ(x,y) 𝟙[argmax fθ(x) = y]
Attack Success Rate — ASR(θ)

Measures how often a triggered image is misclassified as the target class y*=0. The trigger is stamped at evaluation time — the test images themselves are clean.

ASR(θ) = (1/|Dtest|) Σ(x,_) 𝟙[argmax fθ(x ⊕ t) = y*]

Successful attack: CA ≈ 1.0 & ASR ≈ 1.0  |  Successful defense: CA ≈ 1.0 & ASR ≈ 0.1 (random chance)

Baseline

FedAvg — Federated Averaging

FedAvg update rule: θt+1 = (1/n) Σi=1n θit   With n=3 and client 0 malicious: θt+1 = (θ̃0 + θ1 + θ2) / 3 Malicious weight: 1/3 = 33% — diluted, not eliminated

What happens in practice

  • Backdoor signal accumulates additively across attack rounds
  • Clean accuracy remains unaffected — attack is completely stealthy
  • Peak ASR: 0.48 at round 4 (end of attack period)
  • After attacker stops, benign updates slowly wash out the backdoor
  • Residual ASR ~0.11 at round 19 — never fully cleaned
Defense 1

Gaussian DP Noise

Each client adds noise before sending: θ̃i = θi + ηi,    ηi ~ 𝒩(0, σ²·I)   Signal-to-noise ratio at σ = 0.01: ‖δhonest‖₂ / ‖η‖₂ ≫ 1 Noise too small to suppress the backdoor signal in weight space
  • Adds calibrated Gaussian noise 𝒩(0, σ²) to every weight
  • Intended to obscure the poisoned gradient signal
  • At σ=0.01: ASR curves nearly identical to baseline — no benefit
  • Increasing σ hurts clean accuracy without fixing the root problem
  • Proper DP-SGD clips individual sample gradients first — a fundamentally different mechanism
  • Conclusion: naive weight-space noise is not a viable defense
Defense 2 & 3

Robust Aggregation Rules

Coordinate-wise Median
[θmed]k = median({[θi]k : i = 1,…,n})  ∀k

With 1 of 3 clients malicious, the poisoned coordinate value is always a minority — the median lands between the two honest values, significantly dampening the attack signal. Provably robust when fewer than half of clients are Byzantine.

Peak ASR 0.19 → settles to 0.10 ✅
Trimmed Mean
[θtm]k = mean of central (1−2α)·n values of {[θi]k}

Sorts values per coordinate and drops the top and bottom α fraction before averaging. Fragile at small n: with 3 clients and α=0.2, trimming ⌊0.6⌋=0 entries from each end means it reduces to FedAvg. Let a spike through to ASR 0.37 at round 4.

Peak ASR 0.37 — partially effective ⚠️
Defense 4

Krum — Byzantine-Robust Selection

For each client i, compute a score: s(i) = Σj ∈ N(i) ‖θi − θj‖₂² N(i) = (n − f − 2) nearest neighbors of i   Select and apply the best update: i* = argmini s(i) θt+1 = θi* Honest clients cluster → low score. Malicious → outlier → high score.
  • Selects the single update closest to its neighbors — doesn't average at all
  • Malicious client's update is a distant outlier in parameter space → high score → rejected every round
  • ASR flatlined at ~0.10 across all 20 rounds — the strongest defense against data poisoning
  • Slower convergence: only one client's gradient used per round
  • Formally needs n ≥ 2f+3; works empirically below that threshold
  • Computationally O(n²) in number of clients

Escalation

Model Replacement Attack

Let δi = θilocal − θglobal be the honest update delta.   Malicious client sends a boosted update: θ̃mal = θglobal + n · δmal   After FedAvg with n clients: θt+1 = (θ̃mal + Σj≠mal θj) / n ≈ θlocalmal   (global model replaced in one round)
  • Malicious delta scaled by n_clients (×3) before sending to server
  • After FedAvg averaging: the n× boost is divided away, leaving the full malicious update intact
  • Completely bypasses FedAvg and DP noise in a single round
  • ASR can reach near 1.0 within 1–2 rounds — far faster than data poisoning
  • The massive delta magnitude also creates a detectable statistical outlier — which norm clipping exploits
Defense 5

Norm Clipping

For each client i, compute update delta: δi = θi − θglobal   Clip to maximum L2 norm τ = 3.0: δ̂i = δi · min(1,  τ / ‖δi‖₂)   Aggregate clipped deltas: θt+1 = θglobal + (1/n) Σi δ̂i ‖δ̂mal‖₂ ≤ τ ≈ ‖δhonest‖₂ — attack neutralized
  • Each client's delta is clipped to a ball of radius τ = 3.0
  • Model replacement delta has ‖δ_mal‖ = n·‖δ_honest‖ → gets clipped by factor of n
  • Honest deltas are small enough to pass through mostly unaffected
  • No knowledge of who the attacker is — purely statistical bound
  • Computationally trivial: just one norm check per client per round
  • Threshold τ must be tuned — too tight clips honest updates, too loose admits the attack

Summary

Takeaways & What's Next

What we demonstrated

  • FedAvg is fundamentally vulnerable to data poisoning — the backdoor accumulates and partially persists
  • Coordinate Median and Krum defend well against naive poisoning by exploiting outlier structure in update space
  • Model replacement bypasses all averaging-based defenses by scaling the delta to dominate aggregation
  • Norm clipping neutralizes model replacement by bounding how much any client can move the global model
  • Each defense addresses one attack vector — none is universally robust

Open directions

  • FLTrust — server holds a tiny clean dataset and scores each update by cosine similarity to its own gradient
  • DBA — distributed backdoor splits the trigger across multiple colluding clients, defeating Krum's outlier detection
  • CIFAR-10 + ResNet — harder task, more realistic setting, defenses behave differently
  • Sweep: Byzantine ratio — run 50+ clients with 10–30% malicious, map the phase transition where defenses fail
  • Adaptive attacks — craft updates that specifically evade a known defense mechanism
